# Contributing to WorkApp Hub

Thank you for contributing to this application hub!

## Adding a New Application

When adding a new application to this hub, please follow these guidelines:

### 1. Directory Structure

Create a new directory under `apps/` with a descriptive name:

```
apps/
└── your-app-name/
    ├── README.md
    ├── src/
    └── ...
```

### 2. Required Files

Each application must include:

- **README.md**: Description, setup instructions, and usage guide
- **Source files**: Organized in appropriate subdirectories
- **Dependencies**: List any dependencies or requirements

### 3. README Template

Use this template for your application's README:

```markdown
# App Name

Brief description of what the app does.

## Features

- Feature 1
- Feature 2
- Feature 3

## Requirements

List any prerequisites or dependencies.

## Installation

Step-by-step installation instructions.

## Usage

How to use the application.

## License

Specify the license if different from the repository license.
```

### 4. Best Practices

- Keep applications self-contained
- Document all dependencies
- Include example usage where applicable
- Test your application before committing
- Keep commit messages clear and descriptive

## Questions?

If you have questions or need help, please open an issue.
